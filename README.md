# ANBU RP Discord Bot

## Что изменилось в этой версии

### 🔧 Исправленные ошибки
1. **Отсутствовали таблицы в базе данных.** `events`, `family_apps`, `promotions`,
   `vacations`, `shop_points`, `shop_history` не создавались в `init_db()` —
   модули заявок, отпусков, магазина и мероприятий падали с ошибкой при первом
   обращении к БД. Таблицы добавлены.
2. **`UPDATE ... ORDER BY ... LIMIT 1` — невалидный SQL для стандартного SQLite.**
   Такие запросы в `family_app.py` (принятие/отклонение/вызов на обзвон заявки)
   вызывали `OperationalError: near "ORDER": syntax error`. Переписаны через
   подзапрос `WHERE id = (SELECT id ... ORDER BY ... LIMIT 1)`.
3. **Неверный (повреждённый) ID роли админа** в `cogs/tickets.py`
   (`ADMIN_ROLE_ID = 15153294166648424100` — на 1 цифру длиннее, чем валидный
   Discord ID). Все ID теперь задаются командами бота, ошибка исчезла сама собой.
4. Мелкое: неиспользуемые/дублирующиеся импорты убраны, проверки прав HR/Admin
   переведены на `await` (так как настройки теперь хранятся в базе).

### ⭐ Главное: одна команда `/config` для настройки всех ID

Раньше все ID ролей/каналов/категорий были константами в коде, и для каждого
сервера нужно было лезть в файлы и менять их руками.

Теперь всё настраивается **одной командой `/config`** — она открывает меню
(дропдаун) со списком всех параметров. Выбираешь параметр → бот показывает
текущее значение и предлагает:
- выбрать роль/канал/категорию через встроенный селектор Discord;
- ввести ID вручную (кнопка «✏️ Указать ID вручную»);
- сбросить настройку (кнопка «🗑️ Сбросить»).

Для параметров-списков (роли, доступные для выдачи; голосовые каналы вне
трекера) показываются два селектора — добавить и убрать элемент из списка.

Команда `/config` доступна только администраторам сервера.

Также есть `/config_show` — выводит сводку по всем параметрам и их текущим
значениям.

### Список всех настраиваемых параметров

| Параметр в меню `/config` | Где используется |
|---|---|
| Роль администратора | везде, как "роль администрации" |
| Роль HR | АФК, рассылки, повышения, магазин, отпуска, тикеты, заявки |
| Роль поддержки (тикеты) | Тикеты — кто видит и отвечает в тикетах |
| Роль Dep Owner | Заявки в семью — кто рассматривает |
| Роль Recruit | Заявки в семью — пинг при новой заявке |
| Роль участника семьи | Заявки в семью — выдаётся при принятии |
| Роль АФК | АФК — выдаётся автоматически |
| Роль 'В отпуске' | Отпуск — выдаётся автоматически |
| Роль VIP (магазин) | Магазин — выдаётся за покупку товара "vip" |
| Категория для тикетов | Тикеты — куда создаются каналы |
| Канал логов тикетов | Тикеты — лог закрытых тикетов |
| Канал логов АФК | АФК — лог уходов/возвращений |
| Канал логов повышений | Заявки на повышение — куда приходят |
| Канал логов отпусков | Отпуска — лог |
| Канал логов войса | Голосовой трекер — лог входов в войс |
| Канал заявок в семью | Заявки в семью — куда приходят анкеты |
| Канал логов заявок в семью | Заявки в семью — лог (зарезервировано) |
| Категория заявок в семью | Заявки в семью — категория (зарезервировано) |
| Роли, доступные для выдачи (HR-команды) | `/role_give`, `/role_remove` — какие роли можно выдавать (список) |
| Голосовые каналы вне трекера | Голосовой трекер — какие каналы игнорировать (список) |

Что не настроено — просто отключено (бот не пингует, не выдаёт роль, не пишет
в лог), но команды продолжают работать.

## Установка

```bash
pip install -r requirements.txt
# создай .env с токеном бота:
# DISCORD_TOKEN=твой_токен
python bot.py
```

⚠️ В архиве ранее был файл `.env` с реальным токеном бота. **Срочно зайди в
Discord Developer Portal и сгенерируй новый токен** (`Reset Token`), если этот
бот уже где-то запускался — токен из .env нельзя хранить в архивах/репозиториях.

## Первая настройка после запуска

1. Пригласи бота на сервер с правами `Manage Roles`, `Manage Channels`,
   `Send Messages`, `Read Message History`, `View Channels`.
2. От имени администратора выполни `/config` и настрой нужные параметры
   из таблицы выше (по очереди — каждый раз открывай команду заново или
   используй то же сообщение, оно живёт несколько минут).
3. Проверь результат: `/config_show`.
4. Для тикетов: зайди в нужный канал → `/ticket_panel`.
5. Для заявок в семью: `/app_panel`.

## Как получить ID вручную (если нужно «Указать ID вручную»)
1. Включи режим разработчика: Настройки Discord → Расширенные → Режим разработчика.
2. ПКМ на канале/роли/категории → "Копировать ID".
